## 2. 離脱・うまくいかない人が陥るパターン

### 2.1 数値で見る「離脱リスク」層

- **離脱リスク該当者**（以下のいずれか）：**約620人**（回答者の約54%）
  - 投稿数が **10以下**
  - 自己評価（10点満点）が **3以下**
  - サービス満足度が **5以下**
- **投稿数0〜5** の回答者は **約100人以上** 存在。
- **講師満足度が低い（1〜4点）** は約47件、**自己評価が低い（1〜4点）** は約183件。

→ 「行動が止まる」「投稿が少ない」「自己評価が下がる」が重なり、そのまま卒業まで続くケースが多い。

### 2.2 アンケート自由記述から見る「離脱要因」5パターン

不満・満足していない点の記述をキーワードで分類した結果です。

| パターン | ヒット数 | 内容の例 |
|----------|----------|----------|
| **① 操作・使い方がわからない** | **162件** | 「使い方すらわからなかった」「誰に聞けばよいかわからなかった」「仕組みがよくわからなかった」「どこで見れるのかわかりづらい」 |
| **② 質問・サポート不足感** | **138件** | 「誰に聞けばよいかわからない」「添削会をもっと」「フィードバックが抽象的」 |
| **③ 一人で進める負担・孤独** | **112件** | 「教材を1人で進めるのが大変で途中でやらなくなった」「クラスで講義会があったらよかった」「クラスメイトとの交流が欲しかった」 |
| **④ ペース・時間感** | **105件** | 「早くてついていけなかった」「トークの回転が早くてついていけなかった」「時間がさけなかった」 |
| **⑤ 教材・視聴の進め方** | **30件** | 「教材視聴を1人で進めるのが大変」「置いてけぼりになってしまう」「進め方がわからず止まってしまっている」 |

**技術的に操作がわからず離脱**や**質問の仕方・相談先がわからず離脱**は、①・②・⑤に強く現れています。  
**定期的に時間をとって一緒に進める場（作業会・講義会）が欲しかった**という声は③・⑤に直結しています。

### 2.3 クレーム事例から見る「離脱・不満」の共通軸

クレーム対応・事例集めの要約から、次のようなパターンが繰り返し出ています。

#### A. 初期の「見えない・わからない」で不安が膨らむ

- **LINE案内が届かない／届いていない** → 放置感 → 返金要求。
- **グループ編成・クラス編成が遅れる** → 「現状が見えない」不安 → 学習意欲低下。
- **ルール・対象条件の説明不足**（例：ランキング掲載条件）→ 「努力が報われない」不満。

→ **入会直後〜数週間の「案内の届き方」「進捗の見える化」**が離脱と直結。

#### B. 講師・サポートへの不満が離脱に直結

- **フィードバックが抽象的**（「やってみて」「調べて」のみ）→ 「何をどうすればいいかわからない」→ 学習効果を感じられず退会・返金検討。
- **1on1の期待と実際のグループ運営のギャップ** → 「個別指導が受けられていない」不満 → 返金要求。
- **講師の返信遅延・対応のばらつき** → 不安・不信 → 講師変更希望や退会。

→ **「具体的に何をすればよいか」を示すフィードバック**と**応答スピード・一貫性**が満足と継続の中心。

#### C. 期待と提供内容のズレ

- **入会時案内と実際のサービス内容の差**（例：ワーホリ系で伸ばせると思ったが現実は厳しい等）→ 振り回された感覚 → 返金要求。
- **講師が複数クラス掛け持ち** → サポート不足感・不安。

→ **入会時の期待調整**と**役割・体制の説明**が重要。

## 3. 成功している人の特徴（アンケートから）

### 3.1 数値で見る「成功」層

- **成功該当者**（満足度≥8 & 自己評価≥6 & 投稿数≥20）：**約270人**（約24%）。
- 平均値（全回答者）：
  - 満足度 約7.2、投稿数 約35、自己評価 約4.9。
- 成功層は **投稿数・自己評価・満足度がともに高い** 集団。

### 3.2 自由記述から見る「成功の理由」

満足度・自己評価・投稿数が高い人の「満足している点」には、次のような表現が繰り返し出ています。

- **クラス制度・仲間**
  - 「クラス制度があって本当によかった」「同じ目標を持った仲間と出会えた」「一人では伸び悩んだ時やめていたと思う」「クラス制じゃなければ頑張れなかった」
- **質問しやすい・伴走**
  - 「担当講師に気軽に質問できる環境」「講師が伴走しながらアカウントを作り上げていく」「手厚いフォローがある」
- **コンテンツ・環境**
  - 「コンテンツが充分」「卒業後も使えるコンテンツがある」「一緒に6ヶ月走り抜くための環境が整っていた」
- **マインド・変化**
  - 「マインドが強くなり前向きになれた」「リアルに人生が変わった」

→ **「一人にしない」「質問できる」「仲間と一緒に進める」** が成功体験の中心。  
離脱しやすい人に不足しているのは、まさにこの「仕組み化された一緒にやる場」と「質問の入口の明確さ」です。

### 3.3 オフ会参加と成果の関係（参考）

- オフ会「1回以上」と回答した層は、「0回」に比べて **平均投稿数・平均自己評価・平均満足度がやや高い** 傾向。
- ただし「オフ会回数」の回答は0回が大半のため、**参加しやすい形で定期開催する**ことで、さらに差が出る可能性があります。

## 4. 防ぐ方法の方向性（定期的な時間・作業会の位置づけ）

データから、次のような施策が「離脱防止」と「成功者の増加」の両方に効くと整理できます。

### 4.1 技術・操作・「どこに何があるか」でつまずかせない

- **入会直後〜1ヶ月**
  - サービス全体の**仕組み（講師／CM／HR／オフ会／グルコン等）を短く繰り返し説明**する。
  - **「何がわからないか」を聞くタイミング**を明示（例：○週目にショート面談、LINEで「わからないこと3つ」を書いてもらう等）。
- **定期的な「操作・使い方」の場**
  - **定期的な作業会・ガイダンス**で「Portalのここを見る」「Discordのここに書く」「提出はここ」を**一緒に画面を見ながら**確認する。
  - 月1回でも「今月の使い方・よくあるつまずき」を短時間で共有するだけでも、「誰に聞けばいいかわからない」を減らせる。

→ **「技術的に操作がわからなくて離脱」** は、**定期的に時間をとって一緒に触る場**でかなり防げる可能性が高いです。

### 4.2 質問ができずに止まらせない

- **質問の入口を明確に**
  - 「わからないことはここに書く」「○日以内に返信」など、**チャネルと期待値**を案内し続ける。
- **作業会の位置づけ**
  - 「質問OKの作業会」として、**カメラオフ・チャットのみOK**でもよいので、**定期的に時間を取って参加する習慣**を作る。
  - 「何をしていてもよいが、一緒に時間を確保する」だけでも、**一人で教材を進めてやらなくなる**層を減らせる（アンケートの「教材を1人で進めるのが大変」に応える）。

→ **「質問ができなくて離脱」** は、**定期的な作業会＋質問しやすいルールの明示**で防ぐ余地が大きいです。

### 4.3 一人にしない・仲間と進める

- **クラス単位の講義会・作業会**
  - アンケートに「クラスで講義会があったらよかった」「クラスメイトとの交流が欲しかった」という声が複数。
  - **定期的な作業会に参加する**ことで、「同じクラスで同じ時間に取り組む」体験を増やし、孤独感を減らす。
- **進捗の見える化**
  - クレームでは「進捗が見えない」「編成が遅れる」が不満に直結。  
  → 作業会やショート報告で「今どこまで進んでいるか」を軽く共有するだけでも、**止まっている人を早めに検知**できる。

### 4.4 講師・サポート側のばらつきを減らす

- クレームの多くが「講師のフィードバックが抽象的」「対応が遅い」「1on1の定義が違った」に集中。
- **フィードバックの型**（例：必ず「何をどう直すか」を1つ書く、見本・スクショを添える）の標準化。
- **作業会**を、講師や運営が「よくあるつまずき」を直接説明する場にすると、**個別の質問ができなくても、共通の疑問は解消**しやすくなります。

## 5. まとめ：離脱しやすい人・成功する人の傾向と、作業会の役割

| 観点 | 離脱・うまくいかない人 | 成功している人 |
|------|------------------------|----------------|
| **サービス理解** | 操作・仕組み・「どこに何があるか」がわからず、説明不足を感じやすい | クラス制度・コンテンツ・伴走を「使い切っている」 |
| **質問・サポート** | 誰に聞けばいいかわからない、質問ができず止まる | 講師に気軽に質問できる・手厚いフォローを実感 |
| **進め方** | 一人で教材を進めてやらなくなる、ペースについていけない | 仲間と一緒に進める・6ヶ月走り抜く環境がある |
| **初期** | 案内が届かない・編成遅れ・現状が見えないで不安が膨らむ | （成功者は初期から参加・利用している傾向） |
| **講師・運営** | フィードバックが抽象的・遅い・期待と違う | 具体的なアドバイス・伴走を評価 |

**定期的に時間をとって作業会に参加する**ことで期待できる効果は、少なくとも次の3点です。

1. **操作・使い方・「どこに何があるか」**を、**定期的に一緒に確認する** → 技術的つまずきでの離脱を減らす。
2. **「質問OKの時間」を明示する** → 質問ができずに止まる人を減らし、フィードバック待ち以外の経路を増やす。
3. **同じ時間に一緒に取り組む** → 一人で進めてやらなくなる・孤独で止まる人を減らす。

あわせて、**入会直後の案内・進捗の見える化**と**講師フィードバックの具体化・標準化**を進めると、クレームに現れている「説明不足」「期待と現実のギャップ」も抑えやすくなります。
